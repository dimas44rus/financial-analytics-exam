# 📊 Инструкция пользователя: Финансовый Аналитик v1.0

Добро пожаловать в веб-сервис анализа расходов и продаж! Ниже приведена подробная инструкция по запуску, авторизации и загрузке пользовательских данных.

---

## 🚀 1. Запуск приложения

Если вы запускаете проект впервые, убедитесь, что активировано виртуальное окружение, и введите команду в терминале:
```bash
streamlit run main.py
```
После этого в браузере автоматически откроется страница сервиса (обычно по адресу `http://localhost:8501`).

---

## 🔐 2. Как войти в аккаунт (Авторизация)

При открытии сайта система попросит вас пройти проверку:
1. Найдите на экране форму **«🔐 Вход в систему»**.
2. Введите **любое имя пользователя** в поле «Логин».
3. Введите **любые символы** в поле «Пароль».
4. Нажмите кнопку **«Войти в аккаунт»**.
*(Для демонстрации на экзамене заложена сквозная авторизация — подходят любые непустые данные).*

---

## 📁 3. Как загрузить свои данные

После успешного входа приложение по умолчанию показывает встроенные тестовые данные. Чтобы переключиться на свои:
1. Посмотрите на **Боковую панель** слева.
2. В блоке **«Источник данных»** переставьте переключатель на пункт **«Загрузить свои данные (CSV)»**.
3. Нажмите кнопку **«Browse files»** и выберите ваш файл с компьютера.

---

## 📊 4. Требования к формату пользовательских данных (CSV)

Чтобы интерактивные графики, метрики и таблицы успешно построились по вашим данным, файл должен соответствовать **трем простым правилам**:

1. **Формат файла**: Строго `.csv` (сохраняйте из Excel как *CSV с разделителями-запятыми*).
2. **Кодировка**: Рекомендуется `UTF-8` для корректного отображения русского текста.
3. **Названия колонок**: В самой первой строке файла (заголовках) **обязательно** должны быть следующие три колонки (соблюдайте регистр букв):

| Название колонки | Что там должно быть | Пример заполнения |
| :--- | :--- | :--- |
| **`Категория`** | Текстовое название товара, отдела или расхода | `Смартфоны`, `Одежда`, `Продукты` |
| **`Сумма`** | Целые числа или числа с точкой (без букв и пробелов) | `45000`, `3500`, `1200` |
| **`Дата`** | Дата операции в формате Год-Месяц-День | `2026-06-01`, `2026-06-24` |

### Пример правильного содержимого файла (если открыть его в Блокноте):
```text
Категория,Сумма,Дата
Смартфоны,45000,2026-06-20
Одежда,3500,2026-06-21
Продукты,1200,2026-06-22
Смартфоны,52000,2026-06-23
```

*Если вы загрузите файл с другими именами колонок, приложение выдаст предупреждение и продолжит стабильно работать на демонстрационных данных.*
